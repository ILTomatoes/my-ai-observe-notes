# 同一天看到 3 篇 AI 文章,它们讲的是同一件事

那天刷到 3 篇文章,本来没觉得会有关联。读完之后才发现,它们其实在讲同一件事——AI 从"单核工具"被推到了"多核组织"的位置。3 家是同一个范式转变的 3 个切入点。

## 3 篇文章,3 个角度

| # | 文章 | 切入点 | 抽象层 |
|---|------|--------|--------|
| 1 | 扣子 3.0 | 项目制容器 + 异构 Agent + SOP 沉淀 | 平台层 |
| 2 | Claude Dynamic Workflow | 4 种打法 + 3 大失败模式 | 打法层 |
| 3 | Multica 最佳实践 | Issue = 上下文,指派即执行 | 执行层 |

读的时候我画了一张对比表,挑几个差异最大的维度:

| 维度 | 扣子 3.0 | Claude Dynamic | Multica |
|------|----------|----------------|---------|
| 抽象层 | 平台项目容器 | 智能体范式转变 | 组织 Issue 系统 |
| 记忆载体 | 项目空间 | 4 种打法 + 干净子上下文 | Issue / 评论 / metadata |
| Agent 谁派发 | **人决定** | **agent 自己决定** | **人指派** |
| 同步性 | 同步(多端) | 半异步(等你 prompt) | **真正异步(可以睡觉)** |
| 适合谁 | 想"装 Agent 房子" | 想"agent 自己长出打法" | 想"工程师团队替代" |

它们不是替代关系,是互补。选谁取决于团队成熟度:想要完全掌控选扣子,想要 agent 自治选 Claude 路线,想要替代工程师团队选 Multica。

## 4 个跨篇关键词

3 篇文章反复出现同一组词:

**1. 多 agent > 单 agent。** 3 篇都认同。扣子的项目空间是"多 Agent 协作的工作室",Claude 那篇直接列了单 agent 的 3 大失败模式(差不多得了、相信自己、走了一半忘了),Multica 抛出一句很重的话:"工具放大个人产出,组织放大协作产出,差一个数量级。"

**2. 共享记忆空间 > agent 窗口。** 扣子用项目空间装目标、成员、Agent、文件、过程产出;Claude 让子 agent 各持干净上下文互不污染;Multica 把"记忆是窗口"和"记忆是组织资产"放在一起做对比。

**3. 人做决策,不降低地位。** 扣子里人设目标、判方向、确认结果;Claude 那篇有个细节,agent 出错时"反对者 Agent"会提醒人;Multica 提了一个身份转变:"操作者(主语是你)→ 指挥者(主语是事)"——人退到组织者位置,价值反而更高。

**4. 范式转变 > 工具升级。** 3 篇都用了类似的措辞,Claude 那篇说"AI 的下一步,可能不是一个更强的'超级员工',而是一个能随时搭起来的'小团队'",Multica 直接用了"范式的断层"这个词。

## 我自己最在意的 1 件事

3 篇都来自不同的公众号,扣子官方、AgenticAI、Lumiscale,都讲同一件事。这不是巧合,是趋势。

但我更在意的是另一件事:如果只学其中一家都不够。学扣子有容器但里面跑什么打法不知道;学 Claude 设计了打法但没地方装;学 Multica 有 issue 系统但"操作者→指挥者"这一步怎么过渡没想清楚。

把 3 层叠在一起,才是完整的图:

- **容器层**(扣子):项目空间、Plan 持久化、资产可迁移
- **打法层**(Claude):分诊 / 汇总 / 反对者 / 淘汰赛 4 种预编排工作流
- **执行层**(Multica):issue / 任务 / 评论、@触发动作、状态机、人只在 in_review 介入

## 行动项

合并 3 篇的 14 条行动项,去重后留 7 条,按优先级排:

**本周:** 设计稿强制加 3 层架构;选一个高频重复任务场景(比如日/周报生成)跑 v0(分诊任务类型 + 反对者查"无来源断言");把 issue / 任务 / 评论定为核心数据模型,不用聊天窗口当主界面。

**本月:** 写"任务值不值得上工作流"的判断标准(结果重要 / 过程长 / 易漏 / 需验证 / 需多视角 / 需并行才上);metadata 字段严格克制,只写"未来跑的人会反复读"的;把"人决策 / 人验收"当强制设计原则,任何 agent 输出都进 in_review。

**下月评估:** Multica 路线值不值得直接采用,避免重复造 issue 系统的轮子。

## 写在最后

那天写完这 3 篇笔记之后,有一个感觉越来越明确:我现在的笔记流程本身就是这套范式的样本。

抓文章自动化完成,观察笔记 1:1 复刻原文,AI 做独立小节的分析补充,commit + push 自动收尾。人做的是判断和编辑,AI 做的是执行和搬运。

如果 1 年后回看,大概是因为"组织时代"这 3 个字第一次在我脑子里变得具体,而不是因为哪个产品上线了。
