# AI 智能体的两条路,和一个不变的东西

今天连看了 3 篇。花叔写了两篇——一篇聊一人公司为什么注定失败(但他不觉得这是坏事),一篇提出 Vibe Business 这个概念。浪子无为评测了刚发布的 Hermes Agent Desktop。

三篇讲的不是同一件事,但读完之后脑子里画出了一张图。

## AI 智能体在分成两条路

我观察到 AI 智能体正在朝两个方向走:

**第一条路:超级智能体。** 一个 agent 把自己做强——学新技能、记住你的习惯、自动总结经验沉淀成可复用的 skills、定时自己跑任务。代表是 OpenClaw 和 Hermes Agent。浪子无为评测的 Hermes Agent Desktop 就是这条路的产品:长期记忆、自动技能生成、自然语言定时任务、20 多个消息平台接入。

**第二条路:多智能体协作。** 一个核心 agent 当指挥,拉起多个子 agent 分工干活。代表是 Coze 的项目空间、Claude Code 的动态工作流、Multica 的 issue 系统。这条路的特色不是单个 agent 有多强,而是多个 agent 怎么组队、怎么共享上下文、怎么让人只在关键节点介入。

两条路不互斥。Hermes Agent 本身既有 skills 和记忆(方向一),也有 sub-agent 和 cron jobs(方向二的影子)。但从产品形态上看,它们在分叉:方向一走"数字员工"路线,方向二走"数字团队"路线。

## 从写代码到做生意

花叔第一篇提出了一个数字:做 App 只占整个产品生命周期的 5%。

这不是理论。他自己做过一个叫小猫补光灯的工具,用 Cursor 一小时写完,冲到了 AppStore 付费榜 Top 1。但从上线那天起,剩下的事情像一座山压下来——用户在评论区问怎么用、每天回到凌晨、小红书 B 站抖音公众号每个平台风格都不一样、变现怎么做得自己想。

Vibe Coding 解决了那 5%。花叔第二篇提出 Vibe Business 这个概念,意思是该管剩下那 95% 了——推广、客服、复购、运营。

他拿一个叫"码上飞"的产品跑了一遍闭环:一句话生成一个带后台的 App,经营助手自动出推广文案和海报,AI 客服 7×24 顶在前面,最后还能一键发布成微信小程序。

但我读完之后的感觉是:码上飞本质上就是给一个 Coze 项目加了输入输出渠道。输入是一句话需求,输出是产品加文案加客服加小程序。能力不新,包装方式新。

## 杠杆的另一头得有东西

花叔两篇文章的底层逻辑是一致的:AI 是杠杆,不是引擎。

这个判断来自 Naval 的公式——specific knowledge 乘以杠杆等于财富。AI 把杠杆做到了极致,但 specific knowledge 那一项如果是零,杠杆再大也是零。

Pieter Levels 一个人做到月收 13.8 万美元,靠的不是 AI,是十年自学的全栈技术栈加对数字游民群体的亲身浸泡。Marc Lou 说"做 20 个产品,知道 19 个会失败",靠的是三年 ship 过 20 多个失败产品练出来的执行肌肉。

AI 让失败从重伤变成擦伤。过去做一件事赔进去 20 万加三年,现在是几千块加几周。试错循环的成本砍到了几乎为零,这是普通人第一次能跑出大公司才有的强化学习。

但"做什么生意""卖给谁""怎么找到那群人"——这三件事,工具解决不了。

## 拼在一起看

把 3 篇拼在一起,是一张完整的图:

方向一(超级智能体)让单个 agent 变强,方向二(多智能体协作)让多个 agent 组队。Vibe Coding 解决了产品实现的 5%,Vibe Business 在试着解决剩下 95% 的执行。杠杆越来越长。

但不管走哪条路,不可压缩的部分从来没变过:

人负责设计。方向一要人设计 agent 的能力边界(装什么 skills、记什么记忆),方向二要人设计协作流(工作流怎么编排、谁派发谁、什么状态才需要人介入)。做生意要人判断做什么、卖给谁、怎么找到那群人。

技术再发达,总得有人拍板"做这个"。AI 让拍板的代价从重伤变成擦伤,但拍板这个动作本身,始终是人的。

## 写在最后

一天读 3 篇,最明确的一个感受是:AI 智能体的产品形态在加速分化,但"人负责判断"这件事,从 6 月 3 号到 6 月 4 号,两天读了 6 篇文章,没有任何一篇能反驳掉。

反而越来越确定。
