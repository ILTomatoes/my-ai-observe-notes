# AI 智能体的两条路,和一个不变的东西

今天连看了 3 篇。花叔写了两篇——一篇聊一人公司为什么注定失败(但他不觉得这是坏事),一篇提出 Vibe Business 这个概念。浪子无为评测了刚发布的 Hermes Agent Desktop。

三篇讲的不是同一件事,但读完之后脑子里画出了一张图。

## 我的观察

### AI 智能体在分成两条路

**第一条路:超级智能体。** 一个 agent 把自己做强——学新技能、记住你的习惯、自动总结经验沉淀成可复用的 skills、定时自己跑任务。代表是 OpenClaw 和 Hermes Agent。

**第二条路:多智能体协作。** 一个核心 agent 当指挥,拉起多个子 agent 分工干活。代表是 Coze 的项目空间、Claude Code 的动态工作流、Multica 的 issue 系统。特色不是单个 agent 有多强,而是多个 agent 怎么组队。

两条路不互斥。Hermes Agent 本身既有 skills 和记忆(方向一),也有 sub-agent 和 cron jobs(方向二的影子)。但从产品形态上看,它们在分叉:方向一走"数字员工"路线,方向二走"数字团队"路线。

### 从写代码到做生意

花叔做了一个叫小猫补光灯的工具,用 Cursor 一小时写完,冲到了 AppStore 付费榜 Top 1。但从上线那天起,剩下的事情像一座山压下来——用户在评论区问怎么用、每天回到凌晨、小红书 B 站抖音公众号每个平台风格都不一样。

事后估算,做 App 只占整个产品生命周期的 5%。剩下 95% 是推广、客服、复购、运营。

Vibe Coding 解决了那 5%。花叔提出 Vibe Business 这个概念,意思是该管剩下那 95% 了。他拿"码上飞"跑了一遍闭环:一句话生成带后台的 App、经营助手出推广文案海报、AI 客服 7×24 承接、一键发布成微信小程序。

但我的感觉是:**码上飞本质上就是给一个 Coze 项目加了输入输出渠道。** 能力不新,包装方式新。

### 杠杆的另一头得有东西

花叔两篇文章的底层逻辑是一致的:AI 是杠杆,不是引擎。

AI 让失败从重伤变成擦伤。过去做一件事赔进去 20 万加三年,现在是几千块加几周。试错循环的成本砍到了几乎为零。

但"做什么生意""卖给谁""怎么找到那群人"——这三件事,工具解决不了。

## AI 锐评

### 你的"两条路"分类比文章本身有价值

浪子无为在夸 Hermes Agent Desktop 好用,花叔在推销码上飞。但你跳出了产品细节,直接画了一条分界线:超级智能体(一个 agent 变强) vs 多智能体协作(多个 agent 组队)。这个分类框架能覆盖目前几乎所有 AI 协作产品——未来任何新工具出来,你都能快速判断它在哪条路上。

### 但你漏了第三条路:嵌入式 agent

GitHub Copilot 嵌入 IDE,Cursor 嵌入编辑器,飞书 AI 助手嵌入工作流。这些不是"独立的超级智能体",也不是"多 agent 协作系统",而是**嵌入现有工具的 AI 层**。方向一是"你去找 agent",方向二是"agent 组队",方向三是"agent 就在你旁边,你不用主动启动它"。

### "码上飞就是 Coze 加 I/O"这个判断锐利,但可能太武断

你说得对,从技术架构上看,码上飞的底座(一句话生成 App)确实像 Coze 项目空间,推广助手和 AI 客战就是预编排工作流。但**包装方式本身就是产品价值**——Coze 需要你理解"项目 / Agent / 工作流"这些概念,码上飞只需要你说一句话。降低认知门槛本身就是创新。

不过你的核心洞察没错:如果 Vibe Business 只是"已有能力的重新打包",那它的护城河很低。Coze / Claude 随时可以加上 I/O 渠道。

### 花叔的 5% 数字是他亲身经历,不是理论

小猫补光灯冲到 AppStore 付费榜 Top 1,然后 95% 的时间花在推广和客服上——这是**真实案例**,不是假设。你把这个提炼成"产品实现只占商业闭环一小部分",提炼得准。

### 你两天读了 6 篇文章,最确定的一件事

不管走哪条路,不可压缩的部分从来没变过:人负责设计。方向一要人设计 agent 的能力边界,方向二要人设计协作流,做生意要人判断做什么、卖给谁。技术再发达,总得有人拍板"做这个"。

AI 让拍板的代价从重伤变成擦伤,但拍板这个动作本身,始终是人的。
