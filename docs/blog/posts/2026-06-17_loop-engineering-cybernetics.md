# 从一个恒温器到自进化 Agent 团队：Loop Engineering 的控制论根基

读到一篇讲 Loop Engineering 的文章，里面说 agent 的"感知→判断→行动→反馈→调整→继续"循环应该由 agent 自己跑完，人只负责定义规则和验收结果。文章列了五块拼图：调度、知识、连接、协作、记忆。

读完之后跟 AI 做了一轮递进式讨论，从这篇文章出发，一路追到了维纳的控制论、Stafford Beer 的 VSM、Argyris 的双环学习。最后收敛成一个七层框架。记录下来。

---

## 我的观察

### 第一层：最小单元——Agent Loop

不管叫 Codex、OpenClaw 还是 Claude Code，剥掉外壳，核心都是同一个结构：

```
目标 → 规划 → 执行 → 判定 → 未达标 → 回到规划
                           → 达标 → 停
```

这就是控制论里的**负反馈闭环**，跟恒温器同构。恒温器感知温度、对比设定值、决定开关压缩机、再感知温度。Agent 感知当前状态、对比目标、决定下一步动作、再感知结果。结构完全一样，只是"被控对象"从温度变成了"任务完成度"。

没什么神秘的。维纳 1948 年就定义了。

### 第二层：让 Loop 靠谱——Harness Engineering

单个 loop 跑起来容易，跑好很难。一个现实数据：单步成功率 95%，20 步后端到端成功率衰减到 36%。

Harness Engineering 是对单个 loop 的调校工艺——上下文管理、工具系统、执行编排、状态记忆、评估观测、约束恢复。六层全是在做一件事：把 36% 拉回去。

它不是一层新结构。它是对底层 loop 的**优化工程**。类比的话，Harness Engineering 不是设计新空调，是把现有空调的温控精度从 ±3 度调到 ±0.5 度。

### 第三层：让 Loop 自动转——Loop Engineering

人每一轮都手动推 loop 太累。改完代码你说"跑下测试"，看完结果你说"没过再试试"，一来一回半小时，你全程的作用就是按回车。

Loop Engineering 的核心主张：**人退出 loop，只在入口定规则、出口验收结果。**

五块拼图里，只有调度（/loop 和 /goal）是循环驱动的。知识是静态手册，连接是通道能力，协作是团队成员，记忆是状态存储。"Loop Engineering"名字里有 Loop，但准确说它是"以自动循环为核心，搭建完整支撑体系，让人退出执行层"的工程方法。

### 第四层：从恒温器到中央空调——多回路系统

单个 agent loop 是一个恒温器。但 CI 监控是一条回路，PR review 是一条，部署验证是一条，工单同步又是一条。多条回路并行运转、共享知识、共享连接、共享记忆，通过主 agent 协调——这就是中央空调系统。

从 CI + AI 到 CI in AI 再到 Automate Workflow in AI，本质上是同一个递进：

| 层次 | 表述 | 控制论说法 | 生活类比 |
|------|------|-----------|---------|
| 第一层 | CI + AI | 开环辅助 | 手动开关空调 |
| 第二层 | CI in AI | 闭环自治 | 恒温器自动控温 |
| 第三层 | Automate Workflow in AI | 多回路系统 | 中央空调 |

### 第五层：系统长什么样——VSM

讨论到这里碰到了一个关键问题：Loop Engineering 是动态过程（怎么运转），但系统长什么样（需要哪些部件）谁来回答？

答案是 Stafford Beer 的 VSM（可生存系统模型）。VSM 说任何能存活的系统都需要五种子系统：

| VSM | Agent 团队 |
|-----|----------|
| S1 操作 | 子 agent，干活的 |
| S2 协调 | 防冲突，确保子 agent 不互相干扰 |
| S3 控制 | 主 agent，分配任务、检查结果 |
| S4 情报 | MCP 连接的外部信息 |
| S5 政策 | **人**，定 goal、定方向、定边界 |

VSM 不是 Loop Engineering 的一部分。两者**同源不同层**——VSM 是建筑图纸（结构），Loop Engineering 是动线设计（过程）。共享的根基是控制论的反馈回路：VSM 里 S1 到 S5 之间的每条连线都是反馈回路，只是 Beer 没有画出来。

### 第六层：人退到哪了

人退到 S5 不等于断开连接。主 agent 有三条线跟人连着：常规上报（进度、结果）、接收指令（人随时插入新方向）、紧急警报（严重偏差直接拉警报，不走层层上报）。

VSM 里有一个设计叫 Algedonic Alerts——痛觉信号。底层出现严重偏差时，信号不走正常汇报路径，直接跳到最高层。就像手碰到火，神经不经过大脑慢慢分析，直接缩手。

人不需要盯着 loop，但主 agent 会在该找你的时候找你。

### 第七层：系统自己能进化

如果系统不只能执行，还能根据结果改进自己，就进入自进化：

- **个体进化**：单个 agent 改进自己的方法（S1 内部自我反馈，单环学习）
- **团队进化**：主 agent 动态增减成员、调整分工（S3 层面的结构调整）
- **规则进化**：协调机制本身在变（S4→S5→S3 双环回路）

但进化不是无序变异。你知识库里"有限理性"说的——局部理性的加总可能产生整体很差的结果。每个 agent 都在优化自己的局部目标，合在一起可能互相矛盾。所以自进化系统必须有方向约束：人定的 policy 和系统内置的协调机制。进化是有约束的适应，不是失控的演化。

## AI 锐评

### 恒温器这个类比撑了七层

从第一层到第七层，恒温器→中央空调→建筑图纸，这个类比一直在扩展但没有崩。说明控制论的抽象层次足够高——它不是在描述某个具体系统，而是在描述**所有反馈系统的共同结构**。Loop Engineering 恰好踩在了这条线上。

### VSM 和 Loop Engineering 的"同源不同层"是本次讨论最锐利的发现

之前把 VSM 直接塞进 Loop Engineering 的框架里不够严谨。它们回答的是不同问题：VSM 说"系统该长什么样"，Loop Engineering 说"系统该怎么跑"。但它们共享同一个理论根基——反馈回路。这种"同源不同层"的关系在工程里很常见：建筑图纸和动线设计都需要懂力学，但它们不是同一个东西。

### 待深挖的问题

"Automate Workflow in AI" 的安全边界怎么画？代码可以 git revert，发错邮件没有 undo。当 loop 涉及的不只是代码而是整个工作流时，不可逆操作的禁区设计会成为最关键的工程问题。这不是控制论能回答的——控制论管"怎么控"，但"什么不能碰"是伦理和风险管理的领域。
